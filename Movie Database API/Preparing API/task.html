While working on this Movie Database API, consider it as a very simplified version of IMDB. You'll only have movies and people that participate in the movie creation process stored and accessed through API.

So, before jumping to IDE, think about instances you'll have in your application, which attributes they will have and how will they connect with each other. You won't need to implement them all right away, but it's worth seeing the whole picture before committing to it.

Objectives

Now, when you have a general idea of the project, let's start the implementation.

In this stage, you need to create a basic Flask RESTful API using the flask-restful library. Connect an SQLite database. Then, create the Movie table mapping with SQLAlchemy. You can think of many optional attributes this mapping may have, but ensure it has the id, name, year, and genre fields as required.

After that, create GET and POST endpoints for movie at /movies and GET/movies/<id>.

  • The POST /movies endpoint should take a JSON body containing at least the required fields as the input, and then create a new Movie database entry using this data;
  • The GET /movies endpoint should return a list of all the movies stored in the database, accessible by the movies key, containing only movie's id, name, and genre.
  • Finally, the GET /movies/<id> should return detailed information about the movie, i.e., the remaining fields, accessible by the movie key.

Add the drop_all method and call it on start before checking the tests; it would keep the tests clean. This is applicable to all stages.
Examples

Example 1: adding a new movie

http POST http://127.0.0.1:8000/movies

example response:

{"name": "Hypermovie",
"year": 2022,
"genre": "sci-fi"}

Example 2: a list of movies

http GET http://127.0.0.1:8000/movies

example response:

{"movies": [
     {"id": 1,
      "name": "Hypermovie",
      "genre": "sci-fi"}
]}

Example 3: detailed information about a movie

http GET http://127.0.0.1:8000/movies/1

example response:

{"movie":
     {"id": 1,
      "name": "Hypermovie",
      "year": 2022,
      "genre": "sci-fi"}
}
